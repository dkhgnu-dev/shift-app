# PROJECT_RULES - 友達シフト

このプロジェクトは、勤務シフト作成を支援するツールです。

## 大事にすること

- 友達が実際に使えることを優先する。
- 見た目だけでなく、希望休、必要人数、役割、連勤、休みの公平さなどの条件を整理する。
- いきなり全部を自動化しようとせず、まず小さく動く範囲を決める。
- UI、データ構造、自動生成ロジックを混ぜて一気に直さない。

## 危険領域

以下は慎重に扱ってください。

- シフト自動生成ロジック
- 希望休、勤務可能日、必要人数などの条件保存
- DBや保存ファイルの形式変更
- 既存データの削除・上書き
- 大きな依存ライブラリ追加
- ログイン、個人情報、スタッフ名簿

## 途中棚卸の優先

このプロジェクトは途中から苦戦しているため、まず棚卸を優先します。

棚卸で確認すること:

- 最初に作りたかったシフトツールは何か
- 今できていること、できていないこと
- どこで詰まっているか
- 今の構成が目的に合っているか
- 捨てる部分、残す部分
- 次の1サイクルで直す最小範囲

棚卸結果は `docs/handoff/` または `docs/proposals/` に保存してください。
