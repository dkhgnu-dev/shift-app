## 実装内容

### 1. 手動入力セルの優先保護 ＋ 空欄自動作成

- `backend/models.py`: `FixedAssignment(employee_id, day_index, shift_id)` を追加し、`ShiftInput.fixed_assignments` として受け取れるようにした。
- `backend/shift_solver.py`: `fixed_assignments` に含まれる `(employee_id, day_index)` は `model.Add(x[...] == 1)` で強制固定し、求解対象（ソルバーが選ぶ変数）から除外。固定値以外の全シフトはその日について禁止する制約を追加し、固定セルと矛盾しないようにした。
- `frontend/src/App.jsx`:
  - 新規ボタン「空欄自動作成」（`fillBlanks`）を追加。既存の「最適化シフトを生成」（全体再生成）とは別動作で、現在の `generatedResult.matrix` のうち値が入っているセルを全て `fixed_assignments` としてバックエンドへ送信し、空欄セルのみ穴埋めする。
  - `updateCell(i, d, value)`: セルを手動編集した瞬間に自動的に `isFixed: true` を付与（保護）。`－ (未設定)` を選ぶと保護解除され、次回の空欄自動作成の対象に戻る。
  - 保護セルは枠線（`.shift-cell.fixed`）で視覚的に区別。

### 2. 特殊シフト（有休・応援・店長会・研修・勉強会・希望休・公休）

- `backend/models.py` / `shift_solver.py`: `ShiftType.is_special` を追加。特殊シフトは `build_shift_coverage` で時間帯カバレッジを持たず（店舗人数・登録販売者カバレッジ計算から自動除外）、ソルバーが空欄セルへ自動的に割り当てることもない（`fixed_assignments` 経由でのみ設定可能）。
- `frontend/src/App.jsx`: 各セルの `<select>` に「特殊シフト」の `<optgroup>` として7種を追加。有休/応援/店長会/研修/勉強会を選択すると自動的に「勤務時間を調整」モーダルが開き、既定8h（手動で0〜24hに変更可）を個人の出勤日数・合計時間（左側スタッフバッジ）に加算。希望休/公休は時間0・店舗人数0として「休」と同様に扱う。

### 3. 登録販売者（資格者）のスキマ不在検証 ＆ ❌警告

- `analyzeDay(d)`: 当日の登録販売者の実働シフト時間帯（特殊シフト・休は除外）を統合し、営業時間 8:15〜24:00 のうち1分でもカバーされていない区間を `gaps` として算出。
- デスクトップ表: 日付ヘッダーに `❌登販`（該当区間はツールチップで表示）。モバイル表: 日付ナビ直下にバッジ表示。

### 4. 鍵持ち（朝鍵・夜鍵）判別とハイライト ＆ ❌警告

- `frontend/src/App.jsx`: 従業員に `isKeyHolder` を追加（編集モーダルにチェックボックスあり）。
- `analyzeDay(d)` で当日の実働者のうち開始時刻が最も早い者＝開店担当、終了時刻が最も遅い者＝閉店担当と判定し、それぞれに `isKeyHolder` がいるかを判定。
- 該当セルに `key-open`（薄オレンジ `#fff3e0`）／`key-close`（薄緑 `#e8f5e9`）クラスを付与してハイライト。不在時は `❌朝鍵` / `❌夜鍵` を日付ヘッダー（モバイルはバッジ）に表示。

### 5. 左側スタッフ欄のコンパクト化 ＋ 実績バッジ

- ドラッグハンドルを `GripVertical` アイコンから `⋮⋮` のコンパクトテキスト（`.drag-handle-compact`）に変更。
- `computeEmployeeStats(i)`: 特殊シフトの時間も含めて出勤日数・合計時間を都度計算し、`[ 23日 / 194.8h ]` のようなバッジをデスクトップ表・モバイルカード両方の従業員欄に表示。

### 6. 自由時間入力のポチポチタイムピッカー

- `TimePicker` コンポーネントを新規追加。時/分それぞれに▲▼ステッパー（分は30分刻み）と、時（6〜24）・分（00/15/30/45）の数字タップボタンを配置。
- 従業員編集モーダルの「自由時間を入力」と、ルール設定タブの「シフトパターン管理」カスタムパターン追加フォームの `<input type="time">` を置き換えた。

## 既知の制限事項（今回の実装スコープ外・次サイクル候補）

1. 連勤・連休の上限判定ロジック（`shift_solver.py` 内の `work_var` 系）は、特殊シフト（有休等）を引き続き「勤務扱い」として連勤カウントに含めています。実際には出勤していない特殊シフトの日を「休み」として連勤カウントから除外すべきかは業務判断が必要なため、今回は変更していません。
2. 鍵持ちの開店/閉店判定は「その日の実働シフトの中で開始時刻が最も早い/終了時刻が最も遅い人」という単純なロジックです。同時刻に複数名が開店/閉店を担当するケースは正しく判定されますが、「鍵は特定の1人しか持ち出せない」等の運用ルールがある場合は追加の設計が必要です。
3. 登録販売者スキマ警告・鍵持ち警告は現状フロントエンドの表示のみで、バックエンドのwarnings一覧には含まれません（自動生成ロジック自体は登録販売者カバレッジをハード寄りの制約として維持しているため、通常は警告が出ない設計です）。
4. Cycle 1から持ち越しの既知バグ（`localStorage`初期データ問題、シフトパターン管理一覧の自由時間シフト重複表示）は今回のスコープ外のため未着手です。
