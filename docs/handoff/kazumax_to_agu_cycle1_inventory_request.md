# Kazumax to Agu - Cycle 1 途中棚卸依頼

## 目的

このプロジェクトは、友達シフト作成ツールをアグ単体で進めるための共有リポジトリです。
今回は、いきなり実装せず、まず現状を整理する「途中棚卸」をお願いします。

## 作業前に読むファイル

以下を順番に読んでください。

1. `AGENTS.md`
2. `docs/handoff/CURRENT_STATUS.md`
3. `docs/PROJECT_RULES.md`
4. `docs/handoff/WORKFLOW_RULES.md`
5. `docs/FRIEND_TEXTBOOK.md`

読んだら、作業前にチャットで短く報告してください。

```text
読了報告:
- 読んだファイル:
- 現在地:
- 今回やること:
- 追加で読む詳細ルールの有無と理由:
```

## 今回やること

今回は実装ではなく、途中棚卸だけを行ってください。

確認してほしいこと:

- 最初に作りたかったシフト作成ツールは何か。
- 今できていることは何か。
- 今できていないことは何か。
- どこで詰まっているか。
- 今の構成が目的に合っているか。
- `backend/` と `frontend/` の役割がどう分かれているか。
- サーバー、DB、フロントだけ、既存サービス利用など、今後の作り方として何がよさそうか。
- 残す部分、捨てる部分、直す部分はどこか。
- 次の1サイクルで直すなら、どこだけ直すのがよいか。

## 保存先

棚卸結果は、以下のファイルに保存してください。

```text
docs/handoff/agu_cycle1_inventory_report.md
```

改善案がある場合は、以下にも保存してください。

```text
docs/proposals/agu_cycle1_improvement_proposals.md
```

## GitHub反映

作業が終わったら、以下を行ってください。

1. 変更内容を確認する。
2. `docs/handoff/agu_cycle1_inventory_report.md` が保存されていることを確認する。
3. 必要なら `docs/proposals/agu_cycle1_improvement_proposals.md` も保存する。
4. commitする。
5. pushする。

ただし、以下は絶対にcommit/pushしないでください。

- `.env`
- パスワード
- APIキー
- 本物の個人情報
- 本物のスタッフ名簿
- 本物の勤務データ

Git conflictが出た場合は、勝手に解決せず止まって報告してください。

## Dexレビューへ回す条件

棚卸の結果、以下のどれかに当てはまる場合は、Kazumax側Dexレビューに回す前提で報告してください。

- シフト自動生成ロジックの作り直しが必要。
- 保存形式、DB、データ構造の見直しが必要。
- 変更範囲が大きい。
- アグ単体では判断が難しい。
- 友達またはKazumaxの確認が必要。

```text
完了報告:
- 保存したファイル:
- commit/push:
- Dexレビューが必要か:
- 友達に確認してほしいこと:
```
