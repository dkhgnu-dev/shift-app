# Shift-Ag プロジェクト現状棚卸し (CURRENT_STATUS)

## 1. 何を作りたかったのか（目的）
* ドラッグストア等の小売店舗を対象とした、**多段階最適化（条件を考慮した）自動シフト作成プラットフォーム**。
* 登録販売者の配置ルール、希望休、契約日数、連勤制限（社員5連勤/パート4連勤）、全体平均出勤人数の自動算出、±1名以内の厳格制限、曜日ごとの優先順位（1位〜7位）といった複雑な条件をAI（数理最適化ソルバー）が計算し、ボタン一つで最適な1ヶ月のシフトを自動生成する。
* スマホやPCから直感的に操作できるモダンで美しいUI（ガラスモーフィズム、バージョンバッジ `v4.8` 搭載）を提供する。

## 2. 今何ができているのか（実装済み機能）
### バックエンド (FastAPI + OR-Tools)
* Google OR-Tools (CP-SATソルバー) を用いた**多段階（辞書順）最適化 ＋ スラック自動診断エンジン**（`shift_solver.py`）。
* **絶対遵守ルール（ハード制約）：**
  - **希望休の100%絶対厳守**
  - **連勤上限：** 正社員・準社員は最大5連勤（6連勤以上禁止）、パート・アルバイト等は最大4連勤（5連勤以上禁止）
  - **契約日数の遵守**
  - **出勤人数の「平均±1名以内」強制固定制限（v4.8新機能）：** 全員の契約日数合計から算出される1日あたりの平均出勤人数を軸に、全曜日の出勤人数を `平均 - 1名` 〜 `平均 + 1名` の枠内に強制固定（10人対2人等の極端な偏りは物理的に発生しない）。
* **最適配分ルール：**
  - **登録販売者の絶対配置：** 営業時間全時間帯での1名以上配置
  - **1日の出勤上限：** 総従業員数の70%以下制御（少人数店舗の切り上げ補正対応）
  - **曜日順位（1位〜7位）に基づく人員配分（最大1名差 / v4.8新機能）：**
    - 🥇 1位〜2位（上位）： `平均 + 1名`
    - ⚪ 3位〜5位（中位）： `平均 ピッタリ`
    - 🔴 6位〜7位（下位）： `平均 - 1名`
    - 最優先の1位と最低順位の7位であっても、出勤人数の差は必ず枠内に収まり極端な偏りを防止。
  - **月末の抽選会・大抽選会：** 通常月4日間、7月・12月5日間の自動強化
* **100%失敗しないフォールバック処理：** 極端な制約競合時でもエラーで落とさず、安全な基本仮シフトと診断アドバイスを返す。

### フロントエンド (React + Vite)
* バージョン表記バッジ（`v4.8`）のロゴ横追加。
* 全体シフト表、従業員管理、ルール設定（現在適用中の全AIルール一覧可視化）の3タブ構成。
* 生成結果や従業員データの **LocalStorage 保存** 機能。
* ドラッグ＆ドロップおよびスマホ用上下ボタンによる従業員の並び替え。
* スマホ向けレスポンシブデザイン（ハンバーガーメニュー搭載）。
* AI自動診断アドバイスパネル（警告発生時の改善提案表示）。
