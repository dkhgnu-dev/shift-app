# P2 実装指示書: Cycle 2 UI機能追加（タイムピッカーハイブリッド化 & ☀️/🌙 鍵持ちアイコン表示）

## 概要
Cycle 2 の追加改修として、タイムピッカーUIのスマートハイブリッド化（数字ボタン削除 ＋ 矢印ポチポチ ＋ キーボード直打ち）と、カレンダーセル内での鍵持ち（☀️朝鍵 / 🌙夜鍵）のアイコン・枠線カラー表示を実装します。

---

## 1. フロントエンド改修 (`frontend/src/App.jsx` & `frontend/src/index.css`)

### A. タイムピッカーUIの改修 (`.time-picker`)
1. **数字ボタン群の削除**:
   - タイムピッカーの下部に敷き詰められている `6`〜`24` および `00`〜`45` の数字ボタン群 (`.time-picker-quick-row`, `.time-picker-tap`) を削除する。
2. **矢印 ＋ キーボード直打ちのハイブリッド化**:
   - 上部の `↑` `↓` 矢印ボタン (`.time-picker-step`) で 15 分刻みのポチポチ調整を維持する。
   - 中央の時刻表示部分 (`.time-picker-value`) をタップ/クリックすると直接入力可能な `<input>` フィールド（または直打ち可能テキスト）に切り替わり、キーボードで「0930」や「1500」等の直接入力も行えるハイブリッド仕様とする。

### B. カレンダーセル内での ☀️/🌙 鍵持ちビジュアル表示（【重要】鍵持ち権限の判定厳格化）
1. **鍵持ち権限 (`emp.isKeyHolder === true`) の判定厳格化**:
   - 単に「朝一番の出勤者全員」「夜最後の退勤者全員」にアイコンを出すのは誤り。
   - **`emp.isKeyHolder === true`（鍵持ち権限を持つスタッフ）である場合にのみ `☀️` / `🌙` アイコンおよび枠線カラーを表示すること。**
   - 鍵持ち権限を持たない一般スタッフが朝一番や夜最後に出勤していても、`☀️` や `🌙` のアイコンおよび鍵持ち枠線カラーをつけてはならない。
2. **アイコン描画 (`renderCellNode`)**:
   - 朝一番の開錠時間帯に出勤する **鍵持ちスタッフ (`emp.isKeyHolder === true`)** のセル内にのみ **`☀️`（朝鍵アイコン: オレンジ）** を表示する。
   - 夜最後の施錠時間帯に出勤する **鍵持ちスタッフ (`emp.isKeyHolder === true`)** のセル内にのみ **`🌙`（夜鍵アイコン: パープル）** を表示する。
   - 同一の鍵持ちスタッフが朝夜両方の鍵担当となる日は **`☀️🌙`** を表示する。
3. **セル枠線スタイル (`index.css`)**:
   - `.shift-cell.key-open`: 薄オレンジ背景 (`#FFF7ED`) ＋ オレンジ枠線 (`#FDBA74`) （※鍵持ちスタッフのみ適用）
   - `.shift-cell.key-close`: 薄緑背景 (`#F0FDF4`) ＋ 緑枠線 (`#86EFAC`) （※鍵持ちスタッフのみ適用）
   - `.shift-cell.key-open.key-close`: 薄黄色背景 (`#FEF3C7`) ＋ ゴールド枠線 (`#FCD34D`)

---

## 2. 検証内容
1. `npm --prefix frontend run build` がエラーなく成功すること。
2. ブラウザ上で、ルール設定等のタイムピッカーから数字ボタンが消え、矢印と直打ちの両方で時間設定できること。
3. 鍵持ち権限を持つスタッフ (`isKeyHolder: true`) のみ、朝鍵の `☀️` 、夜鍵の `🌙` アイコンが表示され、枠線カラーが適用されること（鍵持ち権限のない一般スタッフには表示されないこと）。
