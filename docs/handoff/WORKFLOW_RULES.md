# WORKFLOW_RULES - 友達シフト連携ルール

## アイ単体の作業フロー

```text
友達/Kazumax -> アイ -> GitHubへpush -> 必要時Kazumax側Dexレビュー -> アイが修正
```

1. アイは作業前に `git pull --ff-only` で最新版を取得する。
2. `AGENTS.md` と `CURRENT_STATUS.md` を読む。
3. 作業内容が危険でないか判断する。
4. 実装または棚卸を行う。
5. 結果を `docs/handoff/` に保存する。
6. 検証内容を書いてcommit/pushする。

## Dexレビューが必要なとき

以下の場合は、Kazumax側Dexにレビューを依頼してください。

- シフト自動生成ロジックに触った
- DB、保存形式、既存データに触った
- 変更範囲が大きい
- 何度直しても思い通りに動かない
- アイが不安を感じる
- Kazumaxまたは友達が「Dexにも見てもらって」と言った

## Dexへ渡す報告に含めること

Dexレビューを依頼する場合、`docs/handoff/` に以下を保存してください。

- 何をしたか
- どのファイルを変更したか
- どう確認したか
- まだ不安な点
- Dexに特に見てほしい点

## Kazumax/Dex側の作業フロー

Kazumax側Dexがこのプロジェクトを見る場合は、必ず作業前に最新版をpullしてください。
レビュー結果や修正指示は `docs/handoff/` に保存し、必要ならcommit/pushしてください。

## 人間確認を減らすルール

人間に「全部確認して」と丸投げしないでください。
確認が必要な場合は、見る場所を1〜5個に絞ってください。

- 確認不要: 文書整理、軽微な内部整理
- 軽い確認: UIや通常導線の小さな変更
- 必須確認: シフト自動生成ロジック、保存、削除、本番利用、個人情報
