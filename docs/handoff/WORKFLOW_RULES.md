# WORKFLOW_RULES.md

## 開発ワークフロー
1. **作業開始時:** `AGENTS.md` および `docs/handoff/CURRENT_STATUS.md` を読み込み、現状と課題を把握する。
2. **作業中:** 要求された修正・追加機能を実装し、必要に応じて検証する。
3. **GitHubへのPush前:** 必ずユーザーに変更内容（概要）を提示し、「GitHubにPushして良いか」の明確な承認を得る。ユーザーの了解なしに自動でPushしてはならない。
4. **作業終了時/棚卸時:** `docs/handoff/CURRENT_STATUS.md` を更新し、「読了報告」「進捗」「今回の取り組み」「Gitコミット/ストップ位置」を報告する。
