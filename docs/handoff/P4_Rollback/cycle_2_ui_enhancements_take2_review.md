## 必須修正（P1）

### P1: 編集中の矢印操作とblur確定が競合する

**修正対象**: `frontend/src/App.jsx` の `TimePicker`。必要ならUIテストまたはTimePicker状態遷移を切り出したテストも追加する。

**問題の再現**:

1. 現在値を`09:00`とする。
2. 中央の時刻を押し、`1500`を入力する。
3. Enterではなく時の上矢印を押す。
4. inputのblurが`15:00`を確定した後、矢印の`onClick`がレンダー時の旧値`09:00`を基準に`10:00`を送る可能性がある。

**期待する状態遷移**:

1. 入力中に矢印を押した場合も、編集中の値を失わない。
2. 方針は、編集中は矢印を無効化する、または入力値を確定してからその確定値へ矢印操作を適用する、のいずれかに統一する。
3. Enter・blurでは有効な4桁時刻だけを確定し、Escape・不正値では編集前の値を保持する。
4. `24:00`保持、時±1、分±15を壊さない。

**受入確認**:

1. `09:00`から`1500`を入力して時の上矢印を押したとき、`10:00`へ巻き戻らない。採用した方針どおり、`15:00`保持・矢印無効、または`16:00`となることを確認する。
2. 分矢印でも同じ競合が起きない。
3. `0930 -> 09:30`、`1500 -> 15:00`、`2400 -> 24:00`、`2430`拒否を確認する。
4. Enter・blurで確定、Escapeで変更なしを実際のUI操作または恒久コンポーネントテストで確認する。
5. 時±1、分±15、24:00保持の既存回帰テストを実行する。

**触らない範囲**: 鍵アイコン・開閉店セル色・バックエンド・シフト保存形式は変更しない。
