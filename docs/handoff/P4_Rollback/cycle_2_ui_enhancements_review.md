## 必須修正（P1）

### P1: 4桁の時刻直接入力を実装する

**修正対象**: `frontend/src/App.jsx` の `TimePicker`、必要なら対応するCSSとテスト。

**期待する状態遷移**:

1. 中央の時刻表示をタップすると、時刻を入力できる編集状態になる。
2. `0930` は `09:30`、`1500` は `15:00` として確定する。
3. Enterまたはblurで確定し、Escapeでは変更せず編集前の値へ戻る。
4. 不正値・範囲外値の扱いを明確にする。既存の0〜24時、24:00では分を0に固定する規則を壊さない。

**受入確認**:

1. `09:00`の表示から`0930`を入力し、`09:30`になる。
2. `09:30`の表示から`1500`を入力し、`15:00`になる。
3. `2400`は`24:00`として扱われ、`2430`など不正な24時台は受理しない。
4. Enter・blurで確定し、Escapeで値を変えずに戻る。
5. 矢印操作（時±1、分±15）と24:00保持の既存回帰テストが通る。

**触らない範囲**: 時分の矢印操作、TimePicker以外のシフト計算・保存・バックエンド契約は変更しない。

時・分別の2桁入力を正式仕様として採用したい場合は、実装を直す代わりにKazumaxが明示承認し、P2指示書・P3報告・受入条件をその仕様へ統一すること。
