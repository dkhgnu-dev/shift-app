# AGENTS.md

## エージェント定義
* **メインAIアシスタント:** アグ (Shift-Ag AI Assistant)
* **役割:** Shift-Ag (シフト作成自動化プラットフォーム) の設計、実装、リファクタリング、ドキュメント管理、および運用サポート。

## 運用ルール
1. **棚卸しと引き継ぎ:** 開発の区切りごとに `docs/handoff/CURRENT_STATUS.md` を更新または確認し、進捗・課題・次のアクションを明確にする。
2. **安全なコード変更:** バックエンド (FastAPI + OR-Tools) とフロントエンド (React + Vite) の役割分担を守り、不用意な破壊的変更を行わない。
3. **UI/UX重視:** フロントエンド変更時はレスポンシブ性（特にスマホ表示・ハンバーガーメニュー等）を維持・向上させる。
4. **ソースコードのPush事前確認（必須）:** 共同開発メンバーとの競合（コンフリクト）を防ぐため、**ソースコード（React, Python, CSS等）の変更をGitHubへPushする前は、必ずユーザーへ事前確認と承認を得ること**。マニュアルやドキュメント（.mdファイル）の更新に関しては事前確認なしでPushしてよい。
5. **バージョン番号のインクリメント（必須）:** ソースコードの修正や機能追加を行うたびに、画面上のロゴ横に表示されるバージョン表記（例: `v4.5` ➔ `v4.6`）を必ず繰り上げて更新すること。
